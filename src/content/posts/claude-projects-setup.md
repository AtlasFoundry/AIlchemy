## Why consistency matters more than cleverness

When you're using AI across multiple projects, the gap between good and great output usually isn't the model — it's the setup. I learned this the hard way after months of re-explaining myself every single session.

Claude Projects changed that. Here's the system I built.

## What a Claude Project actually is

Think of it as a persistent workspace. You give it context once — your voice, your goals, the things it should never do — and it holds that context across every conversation. No more re-briefing. No more "as I mentioned earlier."

## The three things I set up in every project

### 1. A voice brief

This is the single most important document in my project. It covers:

- How I write (Bernadette Jiwa-style: precise, human, one idea at a time)
- Words and phrases I never use
- The tone I'm going for in different contexts (email vs. guide vs. social)

I wrote this brief once, properly, and now every output starts from the right place.

### 2. A prompt library

Every task I do more than twice gets a named prompt. I store these as simple text in the project instructions. When I need one, I call it by name: "Use the PDF extraction prompt" and we're off.

### 3. A context document

This is the stuff Claude needs to know about the project itself. For AIlchemy, that's the brand brief, the content categories, the products I'm building, and any stylistic constraints. It's not a chat message — it's a standing instruction.

## The setup that actually works

Start small. One project, one brief, one week. See what breaks. Then improve it. Don't try to build the perfect system before you've used it.

The best AI setup is the one you actually maintain.
